nginx学习笔记——Beginners_guide

发表于 2026-05-15 分类于 技术笔记
这不是对官方文档的翻译，而是对原文逐段进行知识点的总结。无 AI 代笔。

参考：https://nginx.org/en/docs/beginners_guide.html

Configuration File’s Structure

nginx consists of modules which are controlled by directives specified in the configuration file. Directives are divided into simple directives and block directives. A simple directive consists of the name and parameters separated by spaces and ends with a semicolon (;). A block directive has the same structure as a simple directive, but instead of the semicolon it ends with a set of additional instructions surrounded by braces ({ and }). If a block directive can have other directives inside braces, it is called a context (examples: events, http, server, and location).
nginx 有模块构成，这些模块由配置文件的特定指令来控制。指令分为了普通指令和块指令。
普通指令：名字+【空格】+参数+【英文分号】
块指令：和普通指令类似，但必须用 braces（{}）抱起来。
如果一个块指令当中包含了其他指令，那么就称这个块为一个 context（上下文）。其实就是大括号内的配置的”环境、所处范围”。

Directives placed in the configuration file outside of any contexts are considered to be in the main context. The events and http directives reside in the main context, server in http, and location in server.
配置文件中的指令如果位于其他 contexts 之外，就是 main context。比如 events 和 http 。

The rest of a line after the # sign is considered a comment.
＃ 号之后的内容是注释。

Serving Static Content

An important web server task is serving out files (such as images or static HTML pages). You will implement an example where, depending on the request, files will be served from different local directories: /data/www (which may contain HTML files) and /data/images (containing images). This will require editing of the configuration file and setting up of a server block inside the http block with two location blocks.
Web 服务器的主要功能是提供文件（如图像和静态 html 页面）。这里将示例网站的文件存放在 /data/www 和 /data/images 目录下。
需要编辑配置文件中 http 块中的 server 块，并在 server 块中编辑两个 location 块来指定上述两个目录。

First, create the /data/www directory and put an index.html file with any text content into it and create the /data/images directory and place some images in it.
首先，创建刚刚提到的两个目录，并在 /data/www 目录下创建名为 index.html 的文件（名字写错的话，访问网站内容将是 403 Forbidden），并在 /data/images 目录下放置一些图像。

Next, open the configuration file. The default configuration file already includes several examples of the server block, mostly commented out. For now comment out all such blocks and start a new server block:
在配置文件中注释掉 server 块，并写入：

http {
server {
}
}

Generally, the configuration file may include several server blocks distinguished by ports on which they listen to and by server names. Once nginx decides which server processes a request, it tests the URI specified in the request’s header against the parameters of the location directives defined inside the server block.
配置文件中的 server 块一般由监听的端口（listen）和服务名（server_name）来进行区分。在 nginx 决定由哪个 server 处理请求之后，就会把请求头部中指定的 URI 与 location 指定的参数进行比较。

Add the following location block to the server block:

location / {
root /data/www;
}

This location block specifies the “/” prefix compared with the URI from the request. For matching requests, the URI will be added to the path specified in the root directive, that is, to /data/www, to form the path to the requested file on the local file system. If there are several matching location blocks nginx selects the one with the longest prefix. The location block above provides the shortest prefix, of length one, and so only if all other location blocks fail to provide a match, this block will be used.
这个 location 块的参数是 /，比如直接输入网址，那么就是直接过来匹配这个块。nginx 返回的文件就来自大括号中 root 指定的路径。nginx 会找到所有匹配 URI 前缀的 location，最终选择最长的匹配：

Next, add the second location block:

location /images/ {
root /data;
}

It will be a match for requests starting with /images/ (location / also matches such requests, but has shorter prefix).

The resulting configuration of the server block should look like this:

server {
location / {
root /data/www;
}

location /images/ {
    root /data;
}

}

This is already a working configuration of a server that listens on the standard port 80 and is accessible on the local machine at http://localhost/. In response to requests with URIs starting with /images/, the server will send files from the /data/images directory. For example, in response to the http://localhost/images/example.png request nginx will send the /data/images/example.png file. If such file does not exist, nginx will send a response indicating the 404 error. Requests with URIs not starting with /images/ will be mapped onto the /data/www directory. For example, in response to the http://localhost/some/example.html request nginx will send the /data/www/some/example.html file.
以上就是一个可以正常工作的简短配置了，监听默认端口 80，已经可以在本地进行访问了。
直接访问 localhost，那么 nginx 将返回 /data/www/index.html 文件，如果没有该文件（或者说你文件名写错了），那么大概率返回 403 Forbidden 错误提示（除非开了目录浏览，则会显示文件列表。有些情况也会有 404 Not Found，此处不展开）。
访问 localhost/images/example.png ，但本地没有该文件，则返回 404 Not Found 错误。

To apply the new configuration, start nginx if it is not yet started or send the reload signal to the nginx’s master process, by executing:
要应用新配置的话（如果 nginx 已启动），那么就需要该命令重载：

nginx -s reload

In case something does not work as expected, you may try to find out the reason in access.log and error.log files in the directory /usr/local/nginx/logs or /var/log/nginx.
任何异常情况都可以在 access.log 和 error.log 文件中定位问题。
access.log 存放访问记录；error.log 存放错误记录。

Setting Up a Simple Proxy Server

One of the frequent uses of nginx is setting it up as a proxy server, which means a server that receives requests, passes them to the proxied servers, retrieves responses from them, and sends them to the clients.
Nginx 的常见用途是作为代理服务器，负责接受请求，并传递给被代理的服务器，响应该服务器的请求，并将响应发送给客户端。

We will configure a basic proxy server, which serves requests of images with files from the local directory and sends all other requests to a proxied server. In this example, both servers will be defined on a single nginx instance.
接下来配置一个基础的代理服务器，负责处理来自本地目录的图片请求，并将其他所有请求也发送到被代理服务器上。
本示例中，两个服务器都配置在同一个 nginx 实例上。

First, define the proxied server by adding one more server block to the nginx’s configuration file with the following contents:

server {
listen 8080;
root /data/up1;

location / {
}

}

首先在配置文件中添加一个 server 块来为服务器定义被代理的服务。

This will be a simple server that listens on the port 8080 (previously, the listen directive has not been specified since the standard port 80 was used) and maps all requests to the /data/up1 directory on the local file system. Create this directory and put the index.html file into it. Note that the root directive is placed in the server context. Such root directive is used when the location block selected for serving a request does not include its own root directive.
这个 server 块定义了一个监听8080端口的服务器，所有的请求都映射到 /data/up1 目录，location 里面的根目录会根据它所处上下文的 root 设置 (root /data/up1)来选择自己的工作目录。

不过是有优先级的：location 块的 root>server>http

Next, use the server configuration from the previous section and modify it to make it a proxy server configuration. In the first location block, put the proxy_pass directive with the protocol, name and port of the proxied server specified in the parameter (in our case, it is http://localhost:8080):

server {
location / {
proxy_pass http://localhost:8080;
}

location /images/ {
    root /data;
}

}

接下来使用上一小节的服务器配置，将其修改为代理服务器配置。在第一个 location 中放入 proxy_pass 指令，参数指定代理服务器的协议、名称和端口。
详见于：ngx_http_proxy_module — Module ngx_http_proxy_module

We will modify the second location block, which currently maps requests with the /images/ prefix to the files under the /data/images directory, to make it match the requests of images with typical file extensions. The modified location block looks like this:

location ~ .(gif|jpg|png)$ {
root /data/images;
}

The parameter is a regular expression matching all URIs ending with .gif, .jpg, or .png. A regular expression should be preceded with ~. The corresponding requests will be mapped to the /data/images directory.
接下来修改第二个块，让它匹配的 URI 更具体（使用正则）：~代表规定接下来是正则表达式匹配（并规定了区分大小写），\转义了点号。让location匹配图片格式的文件，映射到 /data/images 目录。详见于：location syntax

When nginx selects a location block to serve a request it first checks location directives that specify prefixes, remembering location with the longest prefix, and then checks regular expressions. If there is a match with a regular expression, nginx picks this location or, otherwise, it picks the one remembered earlier.
当nginx选择一个location来处理请求，会先查看那些指定的“前缀”，记住最长的“前缀”，然后再去查看正则。如果正则匹配成功，就选择正则 location，否则就选择之前记住的最长前缀 location。
“前缀”亦即上一小节配置的类似：/data/images/路径。

The resulting configuration of a proxy server will look like this:

server {
location / {
proxy_pass http://localhost:8080/;
}

location ~ \.(gif|jpg|png)$ {
    root /data/images;
}

}

This server will filter requests ending with .gif, .jpg, or .png and map them to the /data/images directory (by adding URI to the root directive’s parameter) and pass all other requests to the proxied server configured above.
这个服务器将筛选出三个后缀名的文件，自己到 /data/images 目录（root 指令）处理请求(拦截并直接处理)，并将其他所有请求给到上一个配置的被代理服务器。

To apply new configuration, send the reload signal to nginx as described in the previous sections.

There are many more directives that may be used to further configure a proxy connection.

Setting Up FastCGI Proxying

nginx can be used to route requests to FastCGI servers which run applications built with various frameworks and programming languages such as PHP.
nginx 可以用来将请求路由到 FastCGI 服务器。
FastCGI 服务器运行着一些由各种框架和编程语言构建的应用程序。

The most basic nginx configuration to work with a FastCGI server includes using the fastcgi_pass directive instead of the proxy_pass directive, and fastcgi_param directives to set parameters passed to a FastCGI server. Suppose the FastCGI server is accessible on localhost:9000. Taking the proxy configuration from the previous section as a basis, replace the proxy_pass directive with the fastcgi_pass directive and change the parameter to localhost:9000. In PHP, the SCRIPT_FILENAME parameter is used for determining the script name, and the QUERY_STRING parameter is used to pass request parameters. The resulting configuration would be:
FastCGI 服务器最基本的 nginx 配置包括：使用 fastcgi_pass 指令代替 proxy_pass 指令，并使用 fastcgi_param 指令设置传递给 FastCGI 服务器的参数。
接下来将上一节的代理配置为基础，把 proxy_pass 指令替换为 fastcgi_pass 指令，并将参数改为 9000 端口。
在 PHP 中，SCRIPT_FILENAME 参数用于确定脚本名称；QUERY_STRING 参数用于传递请求参数。

这两个参数是 FastCGI 标准定义的环境变量。
Nginx 是参数的传递者；PHP-FPM 是参数的使用者。
这些参数可以在 /etc/nginx/fastcgi_params 文件里查看。

server {
location / {
fastcgi_pass localhost:9000;
fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
fastcgi_param QUERY_STRING $query_string;
}

location ~ \.(gif|jpg|png)$ {
    root /data/images;
}

}

This will set up a server that will route all requests except for requests for static images to the proxied server operating on localhost:9000 through the FastCGI protocol.
这个配置将设置一个路由所有请求的服务器（除了请求静态图像。），这些请求将通过 FastCGI 协议被路由到被代理的后端服务器（localhost:9000）。
