A simple PHP site configuration

一个简单的 PHP 网站配置

接下来给出一个 nginx 如何选择 location 来处理一个经典的 PHP 网站的请求：

nginx 在找 location 时，首先会把前缀字符串记住（比如/），然后再去查看正则匹配，第一个正则被匹配到则停止搜索，nginx 将直接使用这个 location。
如果所有正则都匹配不到，则使用之前找到的最长（最具体）的前缀字符串 location。

需要注意的是，这些匹配只会检查请求行的 URI 部分，不会去看参数（QUERY-STRING）。因为 QUERY-STRING 的参数可以以多种方式来提供，写太多匹配会降低性能（我目前的理解）。

接下来举几个例子说明这个配置的 nginx 会如何处理请求：

  • 请求 “/logo.gif” 首先与 / 前缀的 location 匹配，然后再和 \.(gif|jpg|png)$ 正则匹配，所以最后由后者进行处理。再根据 root 指令，把请求映射到文件 /data/www/logo.gif，并将文件发送给客户端。
  • 请求 “/index.php” 首先也是匹配 location /，再去匹配正则，因此被后者处理。并把请求传递给9000端口的 FastCGI 服务器，fastcgi_param 指令把 FastCGI 参数 SCRIPT_FILENAME 设置为 “/data/www/index.php”，然后 FastCGI 服务器执行该文件。
    • 变量 $document_root 的值等于 root 指令的值
    • 变量 $fastcgi_script_name 的值等于请求 URI
  • 请求 / 更复杂一些。它仅与 location / 匹配。该 location 中的 index 指令将根据参数去 root 下检索文件是否存在。在这个例子中，如果 root 下只有 index.php 文件的话，这个指令会执行一次内部重定向，最后 nginx 会像处理新发来的请求那样重新搜索 location。最后重定向的请求由 FastCGI 服务器处理。

Server names

此文档主要讲 server_name 指令的语法细节（精确名、通配符、正则、优先级等等）。

虚拟主机的名称由 server_name 指令来定义，并指定哪个 server 块来处理请求。
他们可以用精确名称、通配符、正则表达式来命名：

寻找虚拟主机为按照以下优先顺序，在匹配到第一个是定制寻找：

  1. 精确名称
  2. 最长前缀星号通配符名称，比如：*.example.org
  3. 最长后缀星号通配符名称，比如：mail.*
  4. 第一个匹配到的正则表达式名称（按顺序）

注意：
假如配置文件中，正则表达式写在精确名称之前，nginx 也只会在确认完所有精确名称才会去查看正则表达式。
但正则表达式只要匹配到第一个，后面就不会再去查看了。

通配符名称

通配符名称只能在名称的开头或结尾包含星号，且星号必须和点号相邻。例如www.*.example.org 和 w*.example.org 就是无效的。
一个星号可以匹配多个域名片段：*.example.org 不仅匹配 www.example.org，还匹配 www.sub.example.org。

还有一种特殊形式：.example.org 可以匹配精确名称（example.org）和通配符名称（*.example.org）。

否则 nginx 将无法启动，并报错：
