架构组成：Ubuntu 22.04（腾讯云轻量应用服务器）+ Nginx + mysql 8.0 + PHP

阶段 1 项目前置准备与服务器初始化

换源与更新

Ubuntu 默认的官方源在国外，访问速度慢，换成国内源提升速度。我目前租的这台腾讯云服务器默认配好软件源，这里就把已安装的软件包更新到最新稳定版即可。

安装工具

  • wget/curl：文件下载、网络请求工具，后续拉取安装包、验证网站可用性必用
  • net-tools：网络工具包，包含ifconfig、netstat等查看 IP、端口、网络连接的命令
  • chrony：企业级时间同步工具，保障服务器时间准确（日志、备份、监控的基础）
  • ufw：Ubuntu 系统默认的防火墙管理工具，后续安全加固必用

本次项目第一次报错，以下是记录和分析：

核心报错部分：

核心报错，关键词是 EXPKEYSIG —— 这是「GPG 密钥过期」的标准错误标识。

这是因为 MySQL 官方用于签名软件源的 GPG 密钥已过期，本地存储的旧密钥无法通过 apt 的安全校验，导致源被系统自动禁用。

经检验，本地的密钥确实过期半年了。

解决方案（参考链接：MySQL GPG 密钥更新问题解决文档 - 技术栈，【超详细】Ubuntu 系统安装 MySQL 8.0 时 GPG 错误解决方案全攻略（增强版）_mysql8.0没有公钥无法验证签名-CSDN博客）：

为什么选择默认（参考自豆包专家版）：

1. 第一个选项：Use Strong Password Encryption (RECOMMENDED)

  • 底层用的是 caching_sha2_password 认证插件，是 MySQL 8.0 官方主推的强加密认证方式，基于 SHA 256 算法，密码加密强度、防破解能力远高于旧版本，是企业生产环境的强制标准选择。
  • 和后续要做的 mysql_secure_installation 安全初始化完全匹配。

2. 第二个选项：Use Legacy Authentication Method (Retain MySQL 5.x Compatibility)

  • 底层用的是旧版 mysql_native_password 认证插件，是 MySQL 5. X 时代的老旧加密方式，安全性差，官方已经明确不推荐使用。
  • 仅有的适用场景：你需要用十几年前的老旧客户端（比如 PHP 5. X、2018 年之前的老版本 Navicat）连接数据库。

本阶段要安装PHP-FPM，这是 PHP 官方内置的一个 FastCGI 进程管理器。

什么是 CGI？

通用网关接口（Common Gateway Interface），最早的 Web 服务器与后端程序的通信标准/工具，运行在网络服务器上。它的运行逻辑是每来一个 PHP 请求，Web 服务器就临时创建一个 CGI 进程，执行完 PHP 代码后立刻销毁进程。这在处理高并发访问几乎是不可用的，性能极差。

什么是 FastCGI？

它是前者的高性能升级方案，通过预先启动一批 PHP 进程常驻内存来提高效率。Web 服务器通过 FastCGI 协议把请求传给这些常驻进程，处理完后进程不退出，持续等待下一个请求，极大降低了资源开销，提升了并发能力。

** 打通 Nginx 和 PHP-FPM 的通信**：

这一步的目的是让 Nginx 能把. Php 结尾的动态请求，正确转发给 PHP-FPM 处理。

而 Nginx 本身不能直接解析 PHP 代码，它只擅长处理静态资源（图片、HTML、CSS）。而对于 PHP 动态请求，就需要通过 FastCGI 协议来转发给 PHP-FPM 进程，让这一进程来解析执行，执行完成后把结果返回给 Nginx，最终返回给用户。
